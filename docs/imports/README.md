# Imports (Canonical Fixture + Local Working Files)

Use this folder for import datasets and dry-run outputs.

## Canonical repository dataset (Phase 2)
- `members-fixture.xlsx`
- `members-fixture-summary.json`
- `members-fixture-issues.csv`

Do not keep multiple overlapping member fixtures in the repository.

## Local-only files
- Real member spreadsheets (for example `Members.xlsx`) contain personal information and must not be committed.
- Preferred local working name: `members-initial.xlsx`.

## Dry-run naming convention
For the canonical fixture workflow, use:
- `members-fixture-summary.json`
- `members-fixture-issues.csv`
